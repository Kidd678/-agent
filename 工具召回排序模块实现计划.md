# 工具召回排序模块实现计划

## Context

意图分类器 SFT 微调已完成（Qwen2.5-1.5B），接下来构建文档规划中的 **5.3 工具召回排序** 模块。该模块是工具调用链路的核心：用户 query → BGE-M3 向量化 → FAISS 粗排 Top-20 → bge-reranker 精排 Top-5 → 交给 LLM 做 Tool Call 决策。

目标指标：Recall@5 ≥ 89%，MRR ≥ 0.83。

## 需要创建的文件（共 8 个）

### 1. `requirements.txt` — 项目依赖
```
fastapi>=0.111.0
uvicorn[standard]>=0.29.0
pydantic>=2.0.0
openai>=1.30.0
httpx>=0.27.0
faiss-cpu>=1.8.0
FlagEmbedding>=1.2.0
numpy>=1.26.0
python-dotenv>=1.0.0
loguru>=0.7.0
jieba>=0.42.1
rank_bm25>=0.2.2
```

### 2. `app/services/retrieval/__init__.py` — 空文件，包初始化

### 3. `app/services/retrieval/embedder.py` — BGE-M3 编码器
- 单例模式，懒加载 `BAAI/bge-m3`（通过 `FlagEmbedding.BGEM3FlagModel`）
- 通过 `HF_ENDPOINT=https://hf-mirror.com` 环境变量使用镜像源
- `encode_query(text) -> np.ndarray` 编码单条 query
- `encode_documents(texts) -> np.ndarray` 批量编码工具描述
- BGE-M3 默认返回归一化向量，无需额外 L2 归一化

### 4. `app/services/retrieval/faiss_store.py` — FAISS 向量库
- 使用 `faiss.IndexFlatIP`（50 个向量，暴力搜索足够）
- `load(index_dir, registry_path)` 从磁盘加载索引 + 工具元数据
- `search(query_vec, top_k=20)` 返回 `[{tool_id, score, tool_meta}]`

### 5. `app/services/retrieval/reranker.py` — BGE Reranker
- 单例模式，懒加载 `BAAI/bge-reranker-v2-m3`（通过 `FlagEmbedding.FlagReranker`）
- `rerank(query, candidates, top_k=5)` 对 FAISS 粗排结果精排

### 6. `app/services/retrieval/pipeline.py` — 异步编排层
- `initialize()` 启动时预加载所有模型和索引
- `retrieve(query, top_k=5)` 异步接口，内部用 `asyncio.to_thread()` 包裹同步计算
- 流程：encode_query → faiss.search(20) → reranker.rerank(5) → 返回工具列表

### 7. `scripts/build_index.py` — 离线索引构建
- 读取 `tools/registry.json`，为每个工具构建复合文本：`"{name}。{description}。示例：{examples}"`
- 用 BGE-M3 编码，构建 FAISS 索引
- 输出到 `data/tool_index/`：`tools.index`、`tool_ids.json`、`tool_texts.json`

### 8. `eval/eval_retrieval.py` — 评测脚本
- 加载 `data/tool_test/retrieval_testset.json`（50 条测试用例）
- 计算 Recall@1/3/5 和 MRR
- 支持 BM25 baseline 对比（jieba + rank_bm25）

## 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 工具文本表示 | `name。description。示例：examples` | examples 桥接口语化 query 和正式描述的词汇鸿沟 |
| FAISS 索引类型 | IndexFlatIP | 50 个向量，暴力搜索微秒级，无需 IVF/HNSW |
| 异步包装 | asyncio.to_thread() | 避免阻塞 FastAPI 事件循环 |
| 模型下载 | HF_ENDPOINT=https://hf-mirror.com | 用户网络需要镜像源 |

## 执行顺序

1. `requirements.txt`
2. `app/services/retrieval/__init__.py`
3. `app/services/retrieval/embedder.py`
4. `scripts/build_index.py` → 运行生成索引
5. `app/services/retrieval/faiss_store.py`
6. `app/services/retrieval/reranker.py`
7. `app/services/retrieval/pipeline.py`
8. `eval/eval_retrieval.py` → 运行验证指标

## 验证方式

```bash
# 1. 构建索引
python scripts/build_index.py

# 2. 运行评测
python eval/eval_retrieval.py

# 期望输出：Recall@5 ≥ 89%, MRR ≥ 0.83
```
