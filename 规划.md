## 1. 项目概述

### 背景

本项目模拟 vivo 蓝心小v"中控 3.0"架构，实现一个手机端场景下的多工具调度助手。用户输入自然语言 query，系统自动完成意图识别、工具召回、subagent 调度、结果聚合，最终返回自然语言响应。

### 核心目标

| 目标 | 对应 JD 要求 |
|------|-------------|
| 用 FastAPI + asyncio 构建异步服务 
| 对 开源小模型 做意图分类 SFT 微调 
| 实现工具库的向量召回 + reranker 精排 
| 实现 MCP Tool Call + subagent 并发调度 

### 量化目标（简历数字）

- 意图分类准确率：zero-shot ~73% → SFT 微调后 ≥92%
- 工具召回 Recall@5：BM25 ~61% → 向量+重排 ≥89%
- 端到端延迟：串行执行 ~4s → asyncio 并发 ≤2s

```
用户 query
    │
    ▼
┌─────────────────────────────────────┐
│         FastAPI 异步接入层           │
│  POST /chat  ·  asyncio 事件循环    │
└──────────────┬──────────────────────┘
               │ asyncio.gather
       ┌───────┴───────┐
       ▼               ▼
  记忆读取         意图分类器
  (session)      (Qwen-7B SFT)
       └───────┬───────┘
               ▼
    ┌──────────────────────┐
    │      智能路由模块      │
    │  intent → 链路映射   │
    └──┬──────┬──────┬─────┘
       │      │      │
       ▼      ▼      ▼
   工具链路  问答链路  闲聊链路
       │
       ▼
┌─────────────────────────┐
│     工具召回 & 排序      │
│  BGE-M3 → FAISS → Top20 │
│  bge-reranker → Top5    │
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│    MCP Subagent 调度     │
│  Tool Call → 并发执行   │
│  asyncio.gather(tools)  │
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│      响应生成 & 回写     │
│  Qwen-Plus 润色 · 流式  │
│  记忆更新 → session     │
└─────────────────────────┘
```

### 数据流说明

1. 请求进入 FastAPI，`asyncio.gather` 并发发起记忆读取和意图分类
2. 路由模块根据意图类型选择执行链路（工具/问答/闲聊/系统/搜索/导航）
3. 工具链路依次经过召回粗排（FAISS Top-20）→ 精排（reranker Top-5）→ LLM Tool Call 决策
4. `asyncio.gather` 并发执行所有选中工具，结果聚合
5. Qwen-Plus API 对聚合结果润色，流式返回给用户
6. 将本次对话写回记忆模块

## 5. 模块详细设计

### 5.1 FastAPI 异步服务层

**文件：** `app/main.py`、`app/routes/chat.py`

#### 核心设计要点

- 全程使用 `async def`，禁止在异步函数中调用同步阻塞 IO
- 记忆读取和意图分类用 `asyncio.gather` 并发，节省约 300-500ms
- 请求/响应用 Pydantic 严格校验，函数参数全部加类型注解

#### 关键代码结构

```python
# app/routes/chat.py

from fastapi import APIRouter, Depends
from pydantic import BaseModel
import asyncio
from app.services.intent.classifier import IntentClassifier
from app.services.memory.session_store import SessionStore
from app.services.routing.router import Router

router = APIRouter()

class ChatRequest(BaseModel):
    user_id: str
    session_id: str
    query: str

class ChatResponse(BaseModel):
    response: str
    intent: str
    tools_used: list[str]
    latency_ms: float

@router.post("/chat", response_model=ChatResponse)
async def chat(req: ChatRequest) -> ChatResponse:
    import time
    t0 = time.perf_counter()

    # 关键：并发执行，而非串行
    history, intent_result = await asyncio.gather(
        SessionStore.get(req.session_id),
        IntentClassifier.predict(req.query)
    )

    result = await Router.dispatch(
        query=req.query,
        intent=intent_result,
        history=history
    )

    await SessionStore.update(req.session_id, req.query, result.response)

    latency = (time.perf_counter() - t0) * 1000
    return ChatResponse(
        response=result.response,
        intent=intent_result.label,
        tools_used=result.tools_used,
        latency_ms=round(latency, 2)
    )
```

#### 面试考点

- 为什么用 `asyncio.gather` 而不是 `await A; await B`？
  - 答：两个任务没有数据依赖，并发执行节省等待时间；串行时总耗时 = T1+T2，并发时 = max(T1,T2)
- `async def` 和普通 `def` 的区别？协程对象什么时候真正执行？

### 5.4 MCP Subagent 执行层

**文件：** `app/services/agent/`

#### 工具注册与 Tool Call 格式

```python
# app/services/agent/tool_registry.py

from typing import Any

class ToolRegistry:
    """将召回到的工具转换为 OpenAI/Qwen Tool Call 格式"""

    @staticmethod
    def to_openai_tools(tools: list[dict]) -> list[dict]:
        return [
            {
                "type": "function",
                "function": {
                    "name": t["id"],
                    "description": t["description"],
                    "parameters": t["parameters"]
                }
            }
            for t in tools
        ]
```

#### MCP 执行器（核心：并发执行）

```python
# app/services/agent/mcp_executor.py

import asyncio
from openai import AsyncOpenAI
from app.services.agent.tool_registry import ToolRegistry

class MCPExecutor:
    def __init__(self, client: AsyncOpenAI):
        self.client = client

    async def run(self, query: str, tools: list[dict], history: list) -> dict:
        openai_tools = ToolRegistry.to_openai_tools(tools)

        # 第一次调用：让模型决定用哪些工具
        response = await self.client.chat.completions.create(
            model="qwen-plus",
            messages=history + [{"role": "user", "content": query}],
            tools=openai_tools,
            tool_choice="auto"
        )

        message = response.choices[0].message

        if not message.tool_calls:
            return {"response": message.content, "tools_used": []}

        # 关键：asyncio.gather 并发执行所有工具调用
        tool_results = await asyncio.gather(*[
            self._execute_tool(tc.function.name, tc.function.arguments)
            for tc in message.tool_calls
        ])

        # 将工具结果追加到 messages，让模型生成最终回复
        tool_messages = [
            {
                "role": "tool",
                "tool_call_id": tc.id,
                "content": str(result)
            }
            for tc, result in zip(message.tool_calls, tool_results)
        ]

        final_response = await self.client.chat.completions.create(
            model="qwen-plus",
            messages=history + [
                {"role": "user", "content": query},
                message,
                *tool_messages
            ]
        )

        return {
            "response": final_response.choices[0].message.content,
            "tools_used": [tc.function.name for tc in message.tool_calls]
        }

    async def _execute_tool(self, tool_name: str, arguments: str) -> Any:
        """根据 tool_name 路由到对应实现，目前 mock 返回"""
        import json
        args = json.loads(arguments)
        # TODO: 替换为真实工具实现
        return f"[Mock] {tool_name} 执行结果：{args}"
```

### 5.5 记忆管理模块

**文件：** `app/services/memory/session_store.py`

```python
# 简单的内存实现，生产环境可替换为 Redis

from collections import defaultdict, deque
from dataclasses import dataclass

@dataclass
class Turn:
    role: str   # "user" | "assistant"
    content: str

class SessionStore:
    _store: dict[str, deque] = defaultdict(lambda: deque(maxlen=10))  # 最近10轮

    @classmethod
    async def get(cls, session_id: str) -> list[dict]:
        turns = cls._store[session_id]
        return [{"role": t.role, "content": t.content} for t in turns]

    @classmethod
    async def update(cls, session_id: str, query: str, response: str):
        cls._store[session_id].append(Turn("user", query))
        cls._store[session_id].append(Turn("assistant", response))
```

## 6. 数据规划

### 6.1 意图分类数据

| 阶段 | 来源 | 数量 | 用途 |
|------|------|------|------|
| 手写种子 | 人工编写 | 120条（每类20条） | 高质量起点 |
| API 扩写 | Qwen-Plus 扩写 | 480条 | 扩充多样性 |
| 人工校验后 | 筛除错标 | ~500条 | 最终训练集 |
| 测试集 | 独立手写 | 200条（每类~33条） | 评测用，不参与训练 |

**数据划分：** train:dev:test = 8:1:1

**测试集设计要点：**
- 每类包含 5 条"边界 case"（容易混淆的 query）
- 例如"帮我搜一下明天的天气"——应分类为 `tool_call` 而非 `web_search`

### 6.2 工具召回测试集

```json
// data/tool_test/retrieval_testset.json
[
  {
    "query": "帮我设置明天早上8点的闹钟",
    "positive_tool_ids": ["alarm_set"],
    "hard_negative_tool_ids": ["calendar_create", "reminder_set"]
  },
  {
    "query": "记录一下下午3点开会",
    "positive_tool_ids": ["calendar_create"],
    "hard_negative_tool_ids": ["memo_create", "reminder_set"]

**难负样本构造原则：**
- 语义相近但工具不同（闹钟 vs 日历 vs 提醒）
- 领域混淆（导航"去餐厅" vs 搜索"附近餐厅"）
- 共 50 条 query，每条含 1 个正例 + 2-3 个难负例

# training/eval/eval_intent.py
# 指标：Accuracy + 每类 Precision/Recall/F1

from sklearn.metrics import classification_report, accuracy_score

def evaluate(model, test_data):
    y_true, y_pred = [], []
    for item in test_data:
        pred = model.predict(item["input"])
        y_true.append(item["output"])
        y_pred.append(pred)

    print(f"Accuracy: {accuracy_score(y_true, y_pred):.4f}")
    print(classification_report(y_true, y_pred))
```

### 8.2 工具召回评测

```python
# eval/eval_retrieval.py
# 指标：Recall@K（K=1,3,5），MRR

def recall_at_k(results: list[str], positives: list[str], k: int) -> float:
    return int(any(r in positives for r in results[:k]))

def mean_reciprocal_rank(results: list[str], positives: list[str]) -> float:
    for i, r in enumerate(results):
        if r in positives:
            return 1.0 / (i + 1)
    return 0.0
```

**对比实验：**

| 方法 | Recall@5 | MRR |
|------|----------|-----|
| BM25（jieba 分词） | ~61% | ~0.52 |
| BGE-M3 向量检索 | ~81% | ~0.74 |
| BGE-M3 + bge-reranker-v2 | ≥89% | ≥0.83 |

```python
# eval/eval_latency.py

import asyncio
import time

async def measure_latency(n_queries: int = 50):
    queries = load_test_queries(n_queries)

    # 串行执行
    t0 = time.perf_counter()
    for q in queries:
        await sequential_pipeline(q)
    serial_avg = (time.perf_counter() - t0) / n_queries * 1000

    # 并发执行（asyncio.gather）
    t0 = time.perf_counter()
    for q in queries:
        await concurrent_pipeline(q)
    concurrent_avg = (time.perf_counter() - t0) / n_queries * 1000

    print(f"串行平均延迟: {serial_avg:.0f}ms")
    print(f"并发平均延迟: {concurrent_avg:.0f}ms")
    print(f"提升: {serial_avg/concurrent_avg:.1f}x")
```
